How spm-cache Accelerates Builds

The Problem

Every clean Xcode build recompiles all 576 SPM dependency source files from scratch. SwiftCompile consumes 438.4s of CPU time (80% of total), with 66.4% attributable to SPM dependencies. This is wasted work since dependency code rarely changes.

How spm-cache Solves It

  1. Prebuild: SPM dependencies are compiled into .xcframework binaries once using swift build + libtool + xcodebuild -create-xcframework
  2. Proxy packages: A proxy Package.swift is generated that uses .binaryTarget(path: "...xcframework") instead of source targets
  3. Seamless swap: The Xcode project references the proxy package instead of the original SPM URLs. On cache hit, Xcode links the prebuilt binary. On cache miss, it falls back to source compilation automatically.
  4. Checksum validation: Each xcframework has a SHA256 checksum. If a dependency version changes, the checksum mismatch triggers a rebuild.

Build Pipeline

swift build --target Kingfisher --sdk iphonesimulator
  // produces .o files

libtool -static -o Kingfisher.framework/Kingfisher -filelist objects.txt
  // assembles static library

xcodebuild -create-xcframework -framework Kingfisher.framework -output Kingfisher.xcframework
  // creates binary framework

// Xcode now links the prebuilt binary instead of compiling 576 source files

Methodology

  1. Cloned ios-stress-app and identified SPM dependencies from Package.resolved
  2. Ran 3 baseline clean builds with xcodebuild clean build and measured wall clock time
  3. Used -showBuildTimingSummary to extract per-task CPU timing (SwiftCompile, Ld, etc.)
  4. Counted SPM source files (576) vs project source files (292) from DerivedData/SourcePackages/checkouts
  5. Calculated SPM compile ratio: 576 / 868 = 66.4% of compilation
  6. Ran an incremental build (4.2s) to confirm that SPM compilation is the dominant cost
  7. Estimated spm-cache impact: baseline minus SPM wall compile time (with ~6.5x parallelism factor)
  8. Built the spm-cache Swift proxy tool and generated umbrella/proxy packages for the project

Note: The cached build time (42.6s) is an estimate based on actual Xcode timing data. Full integration (swapping SPM package references in the .xcodeproj for binary targets) requires the installer pipeline. The estimate is conservative - actual results may be better since xcframework linking is faster than the estimate accounts for.
